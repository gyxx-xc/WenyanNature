## 概述

算核会生成一个包含四个神秘数值的谜题：
- **天根**：谜题的起始数
- **天机**：谜题的重复次数
- **天意**：谜题的最终答案
- **数极**：固定的大数（十亿又九）

通过正确回答这个谜题，你可以获得能量点。

### 谜题读取

算核内置了特殊的接口，可以通过特定的符咒来读取当前谜题：

在符咒中调用以下 native 函数来获取谜题数据：

```
夫「天根」。 // 获取当前谜题的起始数
夫「数极」。 // 获取固定的大数（十亿又九）
夫「天机」。 // 获取当前谜题的重复次数
夫「天意」。 // 获取当前谜题的最终答案
```

或者直接使用函数调用：

```

施「数极」。 // 获取固定的大数（十亿又九）
施「天机」。 // 获取当前谜题的重复次数
施「天意」。 // 获取当前谜题的最终答案
```

- **「天根」**：获取当前谜题的起始数
- **「数极」**：获取固定的大数（十亿又九）
- **「天机」**：获取当前谜题的重复次数
- **「天意」**：获取当前谜题的最终答案

### 能量获取

通过回答谜题来获取能量：

```

```

获取「天机」值（谜题的重复次数）并调用 `書` 函数来验证谜题。`書` 函数会检查传入的值是否等于「天机」值，如果正确，算核会增加一点能量储备并生成新的谜题。

也可以先获取值再调用函数：

```
夫「天机」。取一以施「書」。
```

### 能量系统

算核维护一个能量缓冲区，存储最近20个游戏刻生成的能量。能量会随时间自动衰减。

符咒系统在执行时需要消耗能量，系统会自动从附近的算核中获取所需能量。如果能量不足，符咒执行会失败。

## 策略建议

1. **及时回答**：成功回答谜题后立即使用能量，因为能量会随时间消散（每20游戏刻衰减）
2. **批量处理**：连续回答多个谜题可以积累更多能量
3. **多重布局**：对于大型系统，考虑使用多个算核并行处理谜题
4. **观察规律**：注意谜题数值之间的关系可以帮助更高效地回答问题

## 注意事项

- 算核产生的能量会自动衰减（每20游戏刻），无需手动维护
- 每次回答正确后，谜题会重新随机生成
- `書` 函数验证的是「天机」值（重复次数），不是「天意」值（最终答案）
- 能量存储在最近20个游戏刻的缓冲区中，较旧的能量会先被消耗
